1)Метод - это то, что может делать объект.Методы можно вызывать и тогда будет происходить прописанное в нем действие
2)Класс - это шаблон для создания объектов.Сам по себе ничего не делает,но позволяет создать объект по готовой структуре
3)Модификаторы доступа:
private: приватный компонент классf. Доступен только в рамках своего класса или структуры.

private protected: компонент класса доступен из любого места в своем классе или в производных классах, которые определены в той же сборке.

file:Класс или структура с такми модификатором доступны только из текущего файла кода.

protected: доступен из любого места в своем классе или в производных классах.Производные классы могут располагаться в других сборках.

internal: компоненты класса или структуры доступен из любого места кода в той же сборке, однако он недоступен для других программ и сборок.

protected internal: совмещает функционал двух модификаторов protected и internal. Такой компонент класса доступен из любого места в текущей сборке и из производных классов, которые могут располагаться в других сборках.

public: публичный, общедоступный компонент класса или структуры. Такой компонент доступен из любого места в коде, а также из других программ и сборок.

4)На данный момент я бы использовал только public и private т.к они самые простые в понимании
5)Для ограничения кода в заданных рамках(private), или наоборот предоставления части кода для все программы(public)

Задание по методам
class Program
{
    static void Main() 
    { 

        int result = Public(10, 5);
        Console.WriteLine($"Результат сложения: {result}");

        Private("Hello, World!");
    }

    public static int Public(int a, int b)
    {
        return a + b;
    }


    private static void Private(string message)
    {
        Console.WriteLine($"Сообщение: {message}");

    }
}

class USER
{
